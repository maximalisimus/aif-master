

auto		Монтируется при загрузке автоматически или после выполнения команды 'mount -a'.
noauto		Файловая система может быть смонтирована только вручную.
exec		Позволяет исполнять бинарные файлы на разделе диска.
noexec		Бинарные файлы не выполняются. Использование опции на корневой системе приведёт к её неработоспособности.
ro			Монтирует файловую систему только для чтения.
rw			Монтирует файловую систему для чтения/записи.
sync		Все операции ввода-вывода должны выполняться синхронно.
async		Все операции ввода-вывода должны выполняться асинхронно.
user		Разрешает любому пользователю монтировать файловую систему. Применяет опции noexec, nosuid, nodev, если они не переопределены.
nouser		Только суперпользователь может монтировать файловую систему.
defaults	Использовать значения по умолчанию. Соответствует набору rw, suid, dev, exec, auto, nouser, async.
suid		Разрешить операции с suid и sgid битами.
nosuid		Запрещает операции с suid и sgid битами.
nodev		На монтируемой файловой системе не будут созданы файлы устройств (/dev). Корневой каталог и целевая директория команды chroot всегда должны монтироваться с опцией dev или defaults.
atime		Включает запись информации о последнем времени доступа (atime) при каждом чтении файла.
noatime		Отключает запись информации о последнем времени доступа (atime) при каждом чтении файла.
relatime	Включает запись информации о последнем времени доступа при чтении файла, если предыдущее время доступа (atime) меньше времени изменения файла (ctime).
norelatime	Отключить relatime;
diratime	Аналогично atime только для каталогов;
dirnoatime	Аналогично noatime, для каталогов;
acl			Включить обработку ACL для раздела.
noacl		Отключить обработку ACL для раздела.
discard		Позволяет использование TRIM, при запросе системы. Trim - сообщает SSD, о блоках, которые можно не хранить физически.
